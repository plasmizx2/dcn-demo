How It Works
Four steps. Zero configuration.
From raw input to aggregated results in seconds. The entire pipeline is automatic.
01

Submit

User submits a job with a task type and input payload via the web dashboard or API.

02

Decompose

The planner analyzes the job and breaks it into independent subtasks optimized for parallel execution.

03

Execute

Worker nodes atomically claim tasks and process them simultaneously across the distributed pool.

04

Aggregate

Results are merged via concatenation or AI synthesis into a single, unified output.

Atomic Task Claiming

PostgreSQL FOR UPDATE SKIP LOCKED ensures no two workers ever claim the same task.

Full Audit Trail

Event sourcing records every state transition — job creation, task splits, worker claims, and completions.

Exponential Backoff

Workers retry with 5s → 15s → 30s backoff. Rate-limited requests trigger doubled backoff.

Async Throughout

FastAPI async endpoints paired with asyncpg connection pooling for non-blocking I/O.
